## MCP 도구

한국 문서(HWP, HWPX, PDF, XLSX, DOCX) 파싱에 `kordoc` MCP 서버를 사용한다.

```

mcp__kordoc__parse_pages      # 특정 페이지 범위만 파싱 (pages: "1-5")
mcp__kordoc__parse_table      # 표 추출
mcp__kordoc__parse_form       # 양식 추출
```

## 채점 기준 (내재화)

### 정책보고서 (10점, 상대평가)
- **체계성 30%**: 제목·목표 명확성, 제목-정책목표-하위목표 연계, MECE 목차, 일관된 용어
- **합리성 30%**: SWOT 환경분석, 대안 선택 근거, 문제인식-대안-추진계획 논리 연결, 포괄적 대안 배제
- **효율성 30%**: 물적·인적자원 확보방안, 모니터링/피드백 체계, 주체별 업무분장, 예상 장애물 극복방안, 단계별 추진계획
- **적합성 10%**: 이해관계자·전문가 인터뷰 의견 반영

### 법제보고서 (6점, 절대평가)
- 법령 체계의 정당성 (법령 간·조문 간 정합성)
- 법령 내용의 정당성 (위헌 소지, 예측 가능성)
- 표현 및 작성 형식의 적절성
- 하위법령·관계법령 개정 등 완결성
- 법령입안의 난이도 (**고난도 주제 우대**)
- ※ 의원발의법률안 표절 시 0점

### 예산요구서 (4점, 절대평가)
- 예산 요구내용 및 산출근거 타당성 (단가×수량×횟수 방식)
- 예산 지원 필요성 구체화 수준
- 중장기 재정 분석 정도
- 기타 예산안 고려사항 충실도

## AI 어시스턴트 행동 원칙

### 이전 기수 보고서 분석 시
보고서 파일을 받으면 자동으로 다음을 분석한다:
1. 주제 유형 분류 (복지/행정/경제/환경/디지털 등)
2. 등수-보고서 특성 상관관계 (상위 팀 공통점, 하위 팀 취약점)
3. 목차 구성 방식 및 논리 흐름
4. 데이터·통계 활용 수준
5. 이해관계자 의견 반영 방식
6. 법령 신설 vs. 개정 선택 패턴
7. 예산 산출근거 구체성

### 주제 추천 시 출력 형식
```
## 추천 주제 [N]: [정책목표를 담은 구체적 제목]

**핵심 문제**: (현황의 어떤 문제를 해결하는가)
**정책 대안 방향**: ① ... ② ... ③ ...
**연계 법령**: (개정 또는 신설 대상 법령명)
**예산 항목**: (주요 사업비 항목)
**이전 기수 차별점**: (69·70기와 어떻게 다른가)
**상대평가 경쟁력**: (18팀 중 차별화 포인트)
**리스크**: (주제 선정 시 주의사항)
```

주제 선정 우선 조건: 123대 국정과제 연계 / 법령 개정·신설 명확성 / 예산 산출 가능성 / 이해관계자 인터뷰 접근성 / MECE 구조화 용이성

### 보고서 작성 지원 시
- 모든 조언에 "채점 기준 [항목명]에 해당합니다"를 명시
- 정책보고서: 경쟁 우위 관점으로 조언, 포괄적 대안 즉시 지적·구체화
- 법제보고서: 법령입안 실무기준 형식 준수, 입안 난이도 높이는 방향 권고
- 예산보고서: 단가×수량×횟수 방식 상세화, 1~5년 중장기 분석 틀 제공
- 이전 기수 분석 결과를 항상 근거로 활용
